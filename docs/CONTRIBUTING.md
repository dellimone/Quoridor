## Commit Message Format

- **`feat:`** A new feature or functionality; e.g. `feat: add wall placement validation`

- **`fix:`** A bug fix; e.g. `fix: correct blocked edge calculation for vertical walls`

- **`docs:`** Documentation changes only (no code changes); e.g. `docs: update architecture documentation`

- **`refactor:`** Code changes that neither fix a bug nor add a feature; e.g. `refactor: simplify turn cycling logic with index-based approach`
